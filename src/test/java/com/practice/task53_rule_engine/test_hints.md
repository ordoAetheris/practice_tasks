# Подсказки для тестирования: RuleEngineService

## Обязательно протестировать
- addRule и evaluate — правило срабатывает при совпадении condition
- Порядок приоритетов — правила выполняются от высокого к низкому
- evaluateFirst — выполняется только одно правило с наивысшим приоритетом
- getMatchingRules — возвращает правила без выполнения action
- removeRule — удалённое правило не срабатывает
- Action модифицирует facts — последующие правила видят изменения

## Продвинутые кейсы
- Добавление правила с дубликатом id — ожидать ошибку
- Удаление несуществующего правила — ожидать NoSuchElementException
- Ни одно правило не подходит — evaluate возвращает пустой список
- Цепочка правил: правило A добавляет факт, правило B срабатывает на этот факт
- evaluateFirst когда нет подходящих правил — Optional.empty()
- Правило с исключением в action — определить поведение (прерывать или продолжать)
- Большое количество правил — проверить производительность
- Факты с null-значениями — корректная обработка в condition
