# Подсказки для тестирования: BTree

## Обязательно протестировать
- Вставка и поиск одного элемента
- Вставка элементов в возрастающем, убывающем и случайном порядке
- Split корня при переполнении (проверить увеличение высоты)
- Каскадный split (вставка, вызывающая разделение на нескольких уровнях)
- Удаление из листа без необходимости перебалансировки
- Удаление из листа с заимствованием у левого/правого брата
- Удаление из листа со слиянием (merge) братьев
- Удаление из внутреннего узла (замена на предшественника/преемника)
- Удаление корня (уменьшение высоты)
- range-запрос: пустой диапазон, один элемент, все элементы
- min/max на пустом и непустом дереве
- size и height корректны после серии вставок и удалений
- inOrderTraversal возвращает отсортированную последовательность

## Продвинутые кейсы
- B-дерево с t=2 (минимальный порядок) — максимум split/merge операций
- B-дерево с большим t (например, t=50) — широкие узлы
- Вставка дубликатов ключей (обновление значения)
- Удаление несуществующего ключа — возвращает false
- Массовая вставка 10 000+ ключей и проверка высоты (должна быть O(log_t(n)))
- Вставка + удаление всех ключей — дерево пусто
- Свойства B-дерева после каждой операции: все листья на одном уровне, ограничения на количество ключей
- Стресс-тест: случайные вставки и удаления с проверкой инвариантов
