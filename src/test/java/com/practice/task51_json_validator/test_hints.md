# Подсказки для тестирования: JsonValidatorService

## Обязательно протестировать
- validate — корректный JSON объект, массив, примитивы
- validate — некорректный JSON (незакрытые скобки, лишние запятые)
- parse объекта — Map с корректными ключами и значениями
- parse массива — List с корректными элементами
- parse примитивов — строки, числа (целые и дробные), boolean, null
- parse вложенных структур — объект в объекте, массив в объекте
- Escape-последовательности в строках — \n, \t, \\, \"
- validateSchema — проверка типов (string, number, boolean)
- validateSchema — обязательные поля (required)

## Продвинутые кейсы
- Пустая строка — validate=false
- Только пробелы — validate=false
- Числа с экспонентой (1e10) — если поддерживается
- Unicode escape (\u0041) — если поддерживается
- Глубоко вложенный JSON (10+ уровней)
- validateSchema с вложенными properties
- Пустой объект {} и пустой массив [] — валидны
- Дублирование ключей в объекте — определить поведение
- Trailing comma {"a":1,} — невалидный JSON
