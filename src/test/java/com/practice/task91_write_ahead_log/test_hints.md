# Подсказки для тестирования: WriteAheadLog

## Обязательно протестировать
- Append и получение записи по LSN
- LSN строго возрастают
- Recovery: committed транзакции восстановлены (redo)
- Recovery: uncommitted транзакции откачены (undo)
- Checkpoint корректно фиксирует состояние

## Продвинутые кейсы
- Несколько транзакций с пересечением по времени
- Сбой после BEGIN без COMMIT/ABORT
- Truncate и последующий recovery
- Повторный recovery (идемпотентность)
