# Подсказки для тестирования: PermissionCheckerService

## Обязательно протестировать
- grant и check — разрешение работает
- deny и check — запрет работает
- Наследование по пути: grant("/org", READ) -> check("/org/team", READ) = true
- deny приоритетнее allow на одном уровне
- Конкретный путь приоритетнее родительского: deny("/org", READ) + grant("/org/team", READ) -> check("/org/team", READ) = true
- Групповые права: grantToGroup -> пользователь группы имеет право
- Индивидуальные права приоритетнее групповых
- getEffectivePermissions — корректный набор прав

## Продвинутые кейсы
- Пользователь в нескольких группах — права накапливаются
- deny на группу + grant на пользователя — grant выигрывает (индивидуальные приоритетнее)
- Корневой путь "/" — распространяется на все ресурсы
- Нет никаких прав для пользователя — check возвращает false (deny by default)
- Глубоко вложенные пути ("/a/b/c/d/e") — наследование работает
- Удаление пользователя из группы — права группы больше не действуют
- Пустой путь или null — ожидать ошибку
