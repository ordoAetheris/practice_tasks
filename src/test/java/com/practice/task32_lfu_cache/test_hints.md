# Подсказки для тестирования: LFUCache

## Обязательно протестировать
- put + get — базовое сохранение и получение
- Вытеснение наименее часто используемого элемента
- LRU tiebreaker — при одинаковой частоте вытесняется наименее недавно использованный
- get увеличивает частоту — часто используемый элемент не вытесняется
- put существующего ключа — обновляет значение и увеличивает частоту
- remove — элемент удалён, size уменьшился
- Capacity <= 0 — ожидать IllegalArgumentException

## Продвинутые кейсы
- Кэш capacity=1: каждый новый put вытесняет предыдущий
- Сценарий: put(1), put(2), get(1), put(3) — вытесняется 2 (freq=1, LRU), а не 1 (freq=2)
- Все элементы с частотой 1 — вытесняется самый старый (LRU)
- Последовательность операций, проверяющая корректность minFrequency
- Удаление элемента с наименьшей частотой — minFrequency пересчитывается?
- get несуществующего ключа — null, без побочных эффектов
- Проверка O(1) сложности — нет циклов по всем элементам в get/put
