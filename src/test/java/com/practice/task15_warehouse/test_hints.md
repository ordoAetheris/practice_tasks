# Подсказки для тестирования: WarehouseService

## Обязательно протестировать
- Приход увеличивает остаток
- Расход уменьшает остаток
- Расход больше остатка — исключение
- История содержит все операции в хронологическом порядке
- Товары с низким остатком

## Продвинутые кейсы
- Несколько приходов и расходов подряд
- Остаток = 0 после полного расхода
- Товар без операций — остаток 0
- getLowStock с разными порогами
- Приход/расход нулевого или отрицательного количества
