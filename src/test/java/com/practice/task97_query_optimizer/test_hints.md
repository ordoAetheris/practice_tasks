# Подсказки для тестирования: QueryOptimizer

## Обязательно протестировать
- Парсинг простого SELECT с условием
- Выбор индекса когда он выгоден
- Порядок JOIN по selectivity
- estimateCost для разных планов
- explain возвращает читаемый план

## Продвинутые кейсы
- Push down predicates через JOIN
- Запрос без индексов — full scan
- Несколько подходящих индексов — выбор лучшего
- Сложный запрос с несколькими JOIN
