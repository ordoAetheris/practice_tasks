# Подсказки для тестирования: ContactDeduplicationService

## Обязательно протестировать
- Добавление контакта — id автоинкрементируется
- Нельзя создать контакт без email и phone одновременно
- Поиск дубликатов по email — два контакта с одинаковым email
- Поиск дубликатов по phone — два контакта с одинаковым телефоном
- Merge — сохраняется контакт с меньшим id, пустые поля заполнены
- Merge — второй контакт удаляется из системы
- Поиск по имени — регистронезависимый
- Поиск по имени — игнорирование лишних пробелов

## Продвинутые кейсы
- Merge когда оба контакта имеют заполненные поля — приоритет у меньшего id
- Merge с самим собой — ожидать корректное поведение или ошибку
- Три контакта с одинаковым email — все в одной группе дубликатов
- Транзитивная дедупликация: A.email == B.email, B.phone == C.phone — одна группа?
- Поиск по имени с пустой строкой
- Удаление несуществующего контакта
- findDuplicates когда дубликатов нет — пустой список
