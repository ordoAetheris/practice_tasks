# Подсказки для тестирования: CronParserService

## Обязательно протестировать
- parse "* * * * *" — каждую минуту, валидно
- parse "*/5 * * * *" — каждые 5 минут, валидно
- parse с некорректным выражением — ожидать IllegalArgumentException
- getNextExecution "0 12 * * *" — следующий полдень
- getNextExecution "0 0 1 1 *" — 1 января следующего года
- getNextN — корректное количество и порядок результатов
- isMatch — true для совпадающей даты, false для несовпадающей
- describe — человекочитаемое описание для базовых выражений

## Продвинутые кейсы
- Списки: "1,15,30 * * * *" — минуты 1, 15, 30
- Диапазоны: "0 9-17 * * *" — с 9 до 17 часов
- Шаг с диапазоном: "0 9-17/2 * * *" — каждые 2 часа с 9 до 17
- День недели: "0 0 * * 1" — каждый понедельник
- Значения вне диапазона (минуты=60, часы=25) — ожидать ошибку
- getNextExecution на границе месяца/года
- 31-е число месяца в месяце с 30 днями
- Выражение "0 0 29 2 *" — 29 февраля (следующий високосный год)
