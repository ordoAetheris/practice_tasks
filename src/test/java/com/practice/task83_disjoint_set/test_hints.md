# Подсказки для тестирования: DisjointSet

## Обязательно протестировать
- makeSet + find возвращает сам элемент как представителя
- union двух множеств — connected возвращает true
- connected для элементов из разных множеств — false
- getComponentSize после нескольких union
- getComponentCount уменьшается после union
- Транзитивность: union(a,b) + union(b,c) => connected(a,c)
- getElements возвращает все элементы компоненты
- makeSet дубликата — исключение
- find/union несуществующего элемента — исключение

## Продвинутые кейсы
- Path compression: после find все узлы указывают на корень (проверить через внутреннее состояние)
- Union by rank: дерево с меньшим рангом присоединяется к большему
- Объединение всех элементов в одно множество — componentCount == 1
- Большое количество элементов (100 000+) — производительность не деградирует
- union(a, a) — множество не меняется, возвращает false
- Строковые элементы, объекты как ключи
- Последовательные union в цепочку: 1-2, 2-3, 3-4, ... — все connected
