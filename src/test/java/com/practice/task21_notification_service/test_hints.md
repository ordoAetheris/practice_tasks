# Подсказки для тестирования: NotificationService

## Обязательно протестировать
- Отправка уведомления — проверить что возвращается объект со статусом SENT
- Пометить как прочитанное — проверить переход SENT -> READ
- Нельзя прочитать уведомление со статусом PENDING или FAILED — ожидать IllegalStateException
- Непрочитанные по пользователю — проверить что возвращаются только SENT
- Пагинация — корректные размеры страниц, выход за границы
- Удаление старых — проверить что удаляются только уведомления старше N дней
- Подсчёт по типу — проверить для разных типов EMAIL/SMS/PUSH

## Продвинутые кейсы
- Пустые userId или title при отправке — ожидать IllegalArgumentException
- Попытка пометить как прочитанное несуществующее уведомление
- Пагинация с пустым списком уведомлений
- Удаление старых когда все уведомления свежие — ожидать 0 удалённых
- Сортировка непрочитанных по дате (новые первыми)
- Повторная отправка FAILED-уведомления
- Подсчёт по типу для пользователя без уведомлений
