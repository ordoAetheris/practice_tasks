# Подсказки для тестирования: TodoListService

## Обязательно протестировать
- Создание задачи — корректные поля, статус TODO, дата заполнена
- Создание с пустым title — ошибка
- Обновление статуса TODO -> IN_PROGRESS -> DONE — успех
- Обновление статуса DONE -> TODO — IllegalStateException
- Удаление существующей и несуществующей задачи
- Фильтрация по статусу — возвращает только задачи с нужным статусом
- Фильтрация по приоритету
- Сортировка: HIGH раньше MEDIUM раньше LOW, внутри приоритета — по дате

## Продвинутые кейсы
- Переход TODO -> DONE напрямую (допустим или нет — зависит от реализации)
- Переход IN_PROGRESS -> TODO (допустим)
- Переход DONE -> IN_PROGRESS (допустим или нет?)
- Множество задач с одинаковым приоритетом — порядок по дате
- Удаление задачи и повторное создание — id не повторяется
