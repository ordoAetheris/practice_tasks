# Подсказки для тестирования: DeliveryTrackerService

## Обязательно протестировать
- Создание посылки — статус CREATED, данные корректны
- Последовательное обновление статусов CREATED -> PICKED_UP -> IN_TRANSIT -> OUT_FOR_DELIVERY -> DELIVERED
- Попытка пропустить статус (например CREATED -> IN_TRANSIT) — ожидать IllegalStateException
- Попытка обновить статус доставленной посылки — ожидать исключение
- Получение текущего статуса после нескольких обновлений
- История отслеживания — все события в хронологическом порядке
- Посылки получателя — фильтрация по имени
- Среднее время доставки

## Продвинутые кейсы
- Переход IN_TRANSIT -> RETURNED (допустимый альтернативный путь)
- Попытка обновить статус возвращённой посылки
- Создание посылки с нулевым или отрицательным весом
- Среднее время доставки при отсутствии доставленных посылок
- Несколько посылок одного получателя в разных статусах
- Проверка location в событиях отслеживания
