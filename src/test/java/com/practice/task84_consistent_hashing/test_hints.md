# Подсказки для тестирования: ConsistentHashingRing

## Обязательно протестировать
- Добавление одного узла — все ключи направляются к нему
- getNode для одного и того же ключа всегда возвращает один и тот же узел (детерминизм)
- Добавление второго узла — часть ключей перераспределяется
- Удаление узла — его ключи переходят к следующему по часовой стрелке
- getNode на пустом кольце — Optional.empty()
- getDistribution — все ключи покрыты, сумма count == количество ключей
- Равномерность распределения при достаточном числе виртуальных узлов

## Продвинутые кейсы
- Минимальность перераспределения: при добавлении 1 узла из N перемещается ~1/N ключей
- Виртуальные узлы: 1 vnode vs 200 vnodes — сравнить стандартное отклонение распределения
- Удаление и повторное добавление узла — распределение восстанавливается
- Добавление дубликата узла — исключение
- Большое количество ключей (100 000) — проверка равномерности
- Стресс-тест: случайное добавление/удаление узлов с проверкой инвариантов
- Разное количество vnodes для разных узлов (weighted distribution)
