# Подсказки для тестирования: CoroutineScheduler

## Обязательно протестировать
- Создание корутины, статус READY
- resume запускает корутину, статус RUNNING -> COMPLETED
- yield приостанавливает, статус SUSPENDED
- schedule выбирает следующую корутину (round-robin)
- Завершённая корутина не возобновляется

## Продвинутые кейсы
- Несколько корутин чередуются через yield
- Корутина завершается без yield
- resume несуществующей корутины
- Порядок выполнения при round-robin
