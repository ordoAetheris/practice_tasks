# Подсказки для тестирования: HotelReservationService

## Обязательно протестировать
- Успешное бронирование свободной комнаты
- Бронирование на пересекающиеся даты — исключение
- Отмена бронирования, комната снова доступна
- Поиск свободных комнат на заданные даты
- Доход за период (только подтверждённые брони)

## Продвинутые кейсы
- Смежные даты (checkOut одной = checkIn другой) — не пересечение
- Несколько комнат, часть занята
- Фильтрация по типу комнаты
- Бронирование на один день
- Доход с учётом отменённых бронирований (не считать)
