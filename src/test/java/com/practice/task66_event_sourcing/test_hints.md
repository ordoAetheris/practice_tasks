# Подсказки для тестирования: EventSourcingStore

## Обязательно протестировать
- append добавляет событие с корректной версией
- getEvents возвращает все события агрегата в порядке версий
- getEvents с диапазоном версий фильтрует корректно
- Optimistic concurrency: конфликт версий бросает исключение
- rebuild восстанавливает состояние из цепочки событий
- getEventsAfter фильтрует по timestamp

## Продвинутые кейсы
- snapshot ускоряет rebuild: проверить что события до снимка не воспроизводятся повторно
- Несколько агрегатов не влияют друг на друга
- Потокобезопасность: параллельные append к одному агрегату — один успешен, другой получает конфликт
- Пустой агрегат (нет событий) — rebuild возвращает пустое состояние
- События неизменяемы (data не модифицируется после append)
- Большое количество событий: rebuild корректен после сотен событий
