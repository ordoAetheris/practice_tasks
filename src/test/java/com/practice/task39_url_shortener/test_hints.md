# Подсказки для тестирования: UrlShortenerService

## Обязательно протестировать
- shorten — возвращает код 6-8 символов из букв и цифр
- resolve — возвращает оригинальный URL
- resolve увеличивает clickCount — проверить через getStats
- deactivate — после деактивации resolve возвращает null
- customShorten — пользовательский код работает
- customShorten с занятым кодом — ожидать IllegalStateException
- Невалидный URL (без http/https) — ожидать IllegalArgumentException

## Продвинутые кейсы
- Один URL — несколько коротких кодов (shorten дважды для одного URL)
- Кастомный код с невалидными символами — ожидать исключение
- Кастомный код слишком короткий (< 3) или длинный (> 20) — исключение
- resolve несуществующего кода — null
- getStats для деактивированного кода — active = false
- Множество shorten — коды уникальны (нет коллизий)
- Статистика после нескольких resolve — clickCount корректен
- Деактивация несуществующего кода — ожидать исключение
