# Подсказки для тестирования: EventSchedulerService

## Обязательно протестировать
- Добавление события — корректные данные
- startTime >= endTime — ожидать IllegalArgumentException
- Конфликты — два пересекающихся события с одинаковым приоритетом
- Нет конфликта — пересекающиеся события с разным приоритетом
- События на дату — фильтрация и сортировка по startTime
- Ближайшие N событий — корректная сортировка
- Свободные слоты — промежутки между событиями

## Продвинутые кейсы
- Повторяющееся DAILY событие — появляется на нескольких датах
- Повторяющееся WEEKLY/MONTHLY событие — проверка генерации
- Свободные слоты при полностью занятом дне — пустой список
- Свободные слоты при пустом расписании — один слот на весь диапазон
- Пересечение повторяющихся событий — обнаружение конфликтов
- Удаление повторяющегося события — все вхождения удалены
- Ближайшие N при N > количества событий
- Пустое имя события — ожидать исключение
