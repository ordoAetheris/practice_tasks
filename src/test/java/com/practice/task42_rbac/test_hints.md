# Подсказки для тестирования: RBACService

## Обязательно протестировать
- Создание роли — проверить корректные поля
- Назначение роли пользователю — hasPermission возвращает true для разрешений роли
- Отзыв роли — hasPermission возвращает false после отзыва
- ADMIN даёт все действия — hasPermission(user, resource, READ/WRITE/DELETE) = true при ADMIN
- Наследование ролей — дочерняя роль имеет разрешения родительской
- Множественные роли — права накапливаются из всех ролей пользователя
- getUserPermissions — возвращает полный набор эффективных разрешений
- getRoleHierarchy — корректная цепочка наследования

## Продвинутые кейсы
- Циклическое наследование ролей — ожидать IllegalArgumentException
- Глубокая цепочка наследования (3+ уровня)
- Назначение несуществующей роли — ожидать ошибку
- Повторное назначение одной роли — идемпотентность
- Отзыв не назначенной роли
- Пользователь без ролей — hasPermission всегда false
- ADMIN на один ресурс не влияет на другие ресурсы
