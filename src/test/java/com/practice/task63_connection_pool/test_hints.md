# Подсказки для тестирования: ConnectionPool

## Обязательно протестировать
- getConnection возвращает валидное соединение
- releaseConnection возвращает соединение в пул для повторного использования
- getPoolSize и getActiveCount корректны после get/release
- Таймаут при getConnection когда пул исчерпан (maxSize достигнут, все заняты)
- shutdown закрывает все соединения
- getConnection после shutdown бросает IllegalStateException
- Невалидные idle-соединения удаляются, создаётся новое

## Продвинутые кейсы
- Конкурентный доступ: множество потоков одновременно запрашивают соединения
- Пул не превышает maxSize при конкурентном доступе
- shutdown ожидает возврата всех активных соединений
- releaseConnection невалидного соединения — оно не возвращается в idle, а удаляется
- Создание пула с maxSize <= 0 бросает исключение
- Нет утечек соединений при исключениях
