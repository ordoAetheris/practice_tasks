# Подсказки для тестирования: StateMachineService

## Обязательно протестировать
- defineState и defineTransition — базовый автомат создаётся
- fire — переход выполняется, getCurrentState возвращает новое состояние
- fire с недопустимым событием — ожидать IllegalStateException
- canFire — true для допустимого перехода, false для недопустимого
- Guard блокирует переход — canFire=false, fire бросает исключение
- Action выполняется при успешном переходе
- getAvailableEvents — корректный список событий
- getTransitionHistory — хронологическая запись всех переходов

## Продвинутые кейсы
- Дублирование перехода (from + event уже определены) — ожидать ошибку
- fire до установки начального состояния — ожидать ошибку
- Цепочка переходов — история записывается корректно
- Guard с изменяющимся контекстом — один раз пропускает, другой нет
- Action модифицирует контекст — изменения видны после перехода
- Определение состояния с дубликатом имени — ожидать ошибку
- Переход в то же самое состояние (self-loop)
