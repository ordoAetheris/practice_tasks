# Подсказки для тестирования: TTLCache

## Обязательно протестировать
- put + get до истечения TTL — значение возвращается
- get после истечения TTL — возвращает null (lazy eviction)
- size возвращает только не истёкшие элементы
- cleanup удаляет истёкшие, возвращает количество удалённых
- remove — элемент удалён
- TTL <= 0 — ожидать IllegalArgumentException
- put с существующим ключом — обновляет значение и TTL

## Продвинутые кейсы
- Элемент с очень большим TTL — не истекает за время теста
- Элемент с маленьким TTL (например 50мс) — Thread.sleep и проверка
- Несколько элементов с разными TTL — одни истекают, другие нет
- size после частичного истечения — корректный подсчёт
- cleanup когда нет истёкших элементов — возвращает 0
- Повторный get истёкшего элемента — null, элемент удалён (не висит в памяти)
- Обновление TTL через повторный put — старый TTL заменяется
