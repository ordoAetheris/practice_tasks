# Подсказки для тестирования: WorkStealingThreadPool

## Обязательно протестировать
- submit одной задачи — результат получен через Future
- submit нескольких задач — все выполнены корректно
- shutdown — новые задачи отклоняются (RejectedExecutionException)
- awaitTermination — возвращает true после завершения всех задач
- Задача с исключением — Future.get() бросает ExecutionException
- getWorkerCount соответствует параметру конструктора
- getQueueSizes — не null, правильная длина

## Продвинутые кейсы
- Work stealing: один worker перегружен, другие помогают (проверить что все задачи выполнены за разумное время)
- Много коротких задач (10 000+) — все выполнены, нет потерянных
- Задачи разной длительности — балансировка нагрузки
- submit после shutdown — RejectedExecutionException
- awaitTermination с таймаутом 0 — немедленный возврат
- Конкурентный submit из нескольких потоков — потокобезопасность
- Пул из одного worker — работает без stealing
- Future.cancel(true) — задача прерывается
- Deadlock-free: задачи, порождающие другие задачи в тот же пул
