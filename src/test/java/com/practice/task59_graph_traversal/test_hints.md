# Подсказки для тестирования: GraphTraversalService

## Обязательно протестировать
- addNode и addEdge — граф строится корректно
- bfs — порядок обхода в ширину (уровень за уровнем)
- dfs — порядок обхода в глубину
- shortestPath — корректный кратчайший путь на известном графе
- hasPath — true для связанных узлов, false для несвязанных
- findCycles — обнаружение цикла (A->B->C->A)
- getConnectedComponents — корректное разбиение на компоненты
- Неориентированное ребро — обход в обе стороны

## Продвинутые кейсы
- Граф из одного узла — bfs/dfs возвращают один элемент
- Ориентированный граф: hasPath(A,B)=true, hasPath(B,A)=false
- shortestPath когда пути нет — пустой список
- shortestPath: из нескольких путей выбирается с минимальным весом
- Отрицательный вес ребра — ожидать IllegalArgumentException
- Добавление ребра с несуществующим узлом — ожидать ошибку
- Дублирование узла — ожидать ошибку
- Граф без рёбер — каждый узел отдельная компонента связности
- findCycles в ациклическом графе — пустой список
- Петля (ребро из узла в себя) — обнаруживается как цикл
