# Подсказки для тестирования: TicketBookingService

## Обязательно протестировать
- Создание мероприятия с секциями — проверить структуру
- Бронирование билета — статус RESERVED, корректное место
- Бронирование уже занятого места — ожидать IllegalStateException
- Оплата в пределах 15 минут — статус PAID
- Оплата после истечения 15 минут — ожидать IllegalStateException
- Отмена билета — статус CANCELLED, место снова свободно
- Свободные места — корректный список после бронирований
- Доход — сумма цен только PAID билетов

## Продвинутые кейсы
- Номер места вне диапазона (0, отрицательный, > totalSeats)
- Оплата уже оплаченного билета
- Отмена уже отменённого билета
- Бронирование после отмены того же места — должно быть доступно
- Доход при отсутствии PAID билетов — ожидать 0.0
- Несколько секций с разными ценами — корректный подсчёт дохода
- Свободные места в полностью забронированной секции — пустой список
