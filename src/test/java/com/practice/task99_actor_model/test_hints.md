# Подсказки для тестирования: ActorSystem

## Обязательно протестировать
- Создание актора и отправка сообщения
- Актор обрабатывает сообщения последовательно (не параллельно)
- ask возвращает Future с результатом
- stop прекращает обработку сообщений
- Dead letters для остановленных акторов

## Продвинутые кейсы
- Supervisor strategy: restart после ошибки
- Supervisor strategy: stop после ошибки
- Цепочка акторов: A -> B -> C
- Высокая нагрузка — mailbox не теряет сообщения
- ask с таймаутом
