# Подсказки для тестирования: LeaderboardService

## Обязательно протестировать
- submitScore — создание нового игрока и обновление существующего
- getTopK — корректный порядок (по убыванию score)
- getTopK — при равном score первый тот, кто получил раньше
- getRank — корректная позиция (1-based)
- getAroundMe — соседи сверху и снизу корректно определяются
- getScoreRange — все игроки в диапазоне включительно
- reset — таблица пуста после сброса

## Продвинутые кейсы
- Отрицательный score — ожидать IllegalArgumentException
- getRank для несуществующего игрока — NoSuchElementException
- getTopK(k) где k больше количества игроков — вернуть всех
- getAroundMe для первого/последнего в рейтинге — корректные границы
- Обновление score — позиция в рейтинге меняется
- Пустая таблица — getTopK возвращает пустой список
- Большое количество игроков — производительность не деградирует линейно
