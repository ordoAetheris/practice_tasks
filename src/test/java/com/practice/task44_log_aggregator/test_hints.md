# Подсказки для тестирования: LogAggregatorService

## Обязательно протестировать
- ingest — запись добавляется корректно
- search по уровню и диапазону — возвращаются только подходящие записи
- search с level=null — возвращаются все записи в диапазоне
- countByLevel — корректные счётчики для каждого уровня
- countBySource — корректное распределение по источникам
- getErrorRate — корректный процент (например, 2 ERROR из 10 = 20.0)
- findPatterns — поиск по regex в message

## Продвинутые кейсы
- Пустое хранилище — search возвращает пустой список, getErrorRate = 0.0
- Логи с одинаковым timestamp — корректная обработка
- getErrorRate с windowMinutes=0 — граничный случай
- findPatterns с некорректным regex — ожидать исключение
- Большой объём логов — проверить что поиск по диапазону эффективен
- countByLevel за период без логов — все счётчики = 0
- Диапазон fromTime > toTime — ожидать пустой результат или ошибку
