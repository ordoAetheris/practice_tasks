# Подсказки для тестирования: PubSubService

## Обязательно протестировать
- createTopic + publish + subscribe: подписчик получает сообщение
- Фильтр: подписчик с фильтром получает только подходящие сообщения
- unsubscribe: после отписки сообщения не доставляются
- deleteTopic: все подписки удаляются
- getMessageHistory возвращает историю с ограничением limit
- Dead Letter Queue: исключение в обработчике → сообщение в DLQ

## Продвинутые кейсы
- Множество подписчиков на один топик — все получают сообщение
- Подписчик без фильтра (null) получает все сообщения
- Потокобезопасность: publish из нескольких потоков одновременно
- Создание дублирующего топика бросает исключение
- Ошибка одного подписчика не влияет на доставку другим
- getSubscribers после unsubscribe — подписчик отсутствует
- Publish в несуществующий топик — обработка ошибки
