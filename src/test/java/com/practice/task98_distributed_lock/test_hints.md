# Подсказки для тестирования: DistributedLockManager

## Обязательно протестировать
- acquire/release одним владельцем
- Второй владелец не может захватить занятый ресурс
- Fencing token строго возрастает
- TTL: блокировка автоматически истекает
- tryAcquire с таймаутом
- Re-entrant: один owner может захватить повторно

## Продвинутые кейсы
- Release с неверным fencing token — исключение
- Refresh продлевает TTL
- Захват после истечения TTL другого владельца
- Несколько ресурсов параллельно
