# Подсказки для тестирования: UndoRedoManager

## Обязательно протестировать
- execute + undo: состояние откатывается
- undo + redo: состояние восстанавливается
- Redo стек очищается при новой команде
- canUndo/canRedo корректны
- undo при пустом стеке бросает исключение
- redo при пустом стеке бросает исключение
- getHistory возвращает все выполненные команды

## Продвинутые кейсы
- executeBatch: undo отменяет все команды группы за один вызов
- executeBatch: redo повторяет все команды группы за один вызов
- maxHistorySize: старые команды удаляются при превышении лимита
- Множественные undo/redo подряд
- clear очищает оба стека
- getUndoStack/getRedoStack корректны после серии операций
- Команда с побочными эффектами: undo действительно откатывает изменение
