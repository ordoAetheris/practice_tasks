# Подсказки для тестирования: VirtualMemoryManager

## Обязательно протестировать
- access: корректная трансляция (pageNumber = virtualAddress / pageSize, offset сохраняется)
- Первый доступ к странице → page fault → загрузка
- Повторный доступ к той же странице → TLB hit (нет page fault)
- write устанавливает dirty=true
- Вытеснение при заполнении всех фреймов
- getPageTable содержит корректные записи
- getTLBHitRate: 0.0 при первых обращениях, растёт при повторных
- getPageFaultCount увеличивается при промахах

## Продвинутые кейсы
- FIFO: последовательность обращений → предсказуемый порядок вытеснения
- LRU: недавно использованные страницы не вытесняются
- Clock (Second Chance): referenced=true даёт второй шанс
- Belady's anomaly для FIFO: больше фреймов → больше page faults (конкретная последовательность)
- TLB eviction при заполнении TLB
- Адрес вне допустимого диапазона → исключение
- Dirty page eviction → должна быть «записана на диск» (отследить)
- Все три политики на одной последовательности обращений → сравнить page fault count
- pageSize не степень двойки → исключение
