# Подсказки для тестирования: LoadBalancerService

## Обязательно протестировать
- RoundRobin: запросы распределяются циклически
- WeightedRoundRobin: сервер с весом 3 получает в 3 раза больше запросов, чем с весом 1
- LeastConnections: выбирается сервер с наименьшим числом активных соединений
- markUnhealthy: нездоровый сервер исключается из выбора
- markHealthy: сервер снова участвует в ротации
- getNextServer при отсутствии здоровых серверов бросает исключение
- getStats корректно считает totalRequests и activeConnections

## Продвинутые кейсы
- RandomWithWeights: при большом количестве запросов распределение пропорционально весам
- removeServer: удалённый сервер не участвует в ротации
- LeastConnections + releaseConnection: после освобождения сервер снова предпочтителен
- Один сервер в пуле — все запросы идут на него
- Потокобезопасность: конкурентные getNextServer
- addServer к существующему пулу — корректная интеграция
- RoundRobin: после markUnhealthy + markHealthy порядок сохраняется
