# Подсказки для тестирования: LRUCache

## Обязательно протестировать
- put + get — базовое добавление и получение
- Вытеснение LRU-элемента при переполнении capacity
- get обновляет порядок — после get элемент не вытесняется следующим
- put с существующим ключом — обновление значения без увеличения size
- remove — элемент удалён, size уменьшился
- size — корректное значение после операций
- clear — кэш пуст, size == 0
- Capacity <= 0 — ожидать IllegalArgumentException

## Продвинутые кейсы
- Кэш с capacity=1 — каждый put вытесняет предыдущий
- Последовательность put, get, put — проверить что get защищает элемент от вытеснения
- Удаление несуществующего ключа — вернуть false, size не изменился
- get несуществующего ключа — вернуть null
- Множество операций — проверить что всегда вытесняется именно LRU
- Проверка что не используется LinkedHashMap (ревью кода)
- put с null значением — допустимо ли?
