# Подсказки для тестирования: RetryMechanism

## Обязательно протестировать
- Успешный вызов с первой попытки — без повторов
- Retryable исключение — повтор до maxAttempts
- Не-retryable исключение — сразу пробрасывается без повтора
- Fixed backoff: задержка одинаковая между всеми попытками
- Exponential backoff: задержка растёт (delay, delay*2, delay*4, ...)
- После исчерпания попыток бросается последнее исключение
- getAttemptHistory содержит все попытки

## Продвинутые кейсы
- maxDelayMs ограничивает задержку сверху
- ExponentialWithJitter: задержка варьируется (не строго экспоненциальная)
- Успех на N-й попытке: предыдущие N-1 были ошибочными
- maxAttempts = 1: без повторов, сразу бросить
- История содержит корректные задержки и timestamps
- Разные типы retryable исключений в одной политике
- Наследник retryable исключения — тоже ретраится (instanceof)
