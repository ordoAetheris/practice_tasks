# Подсказки для тестирования: PlaylistManagerService

## Обязательно протестировать
- Создание плейлиста — проверить что возвращается пустой плейлист с корректным именем
- Добавление трека — проверить что трек появляется в конце списка
- Запрет дублирования трека в одном плейлисте — ожидать IllegalStateException
- Удаление трека — проверить что трек удалён, порядок остальных сохранён
- Перемещение трека — проверить новую позицию и порядок остальных треков
- Общая длительность — сумма durationSeconds всех треков
- Поиск по исполнителю — регистронезависимый, уникальные результаты

## Продвинутые кейсы
- Создание плейлиста с пустым именем — ожидать исключение
- Перемещение трека на текущую позицию — ничего не должно измениться
- Перемещение на невалидную позицию (отрицательную, больше размера)
- Shuffle — проверить что все треки на месте (размер и состав не изменились)
- Общая длительность пустого плейлиста — ожидать 0
- Поиск по исполнителю, который есть в нескольких плейлистах — без дубликатов
- Удаление несуществующего трека из плейлиста
