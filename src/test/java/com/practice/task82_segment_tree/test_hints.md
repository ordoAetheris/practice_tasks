# Подсказки для тестирования: SegmentTree

## Обязательно протестировать
- build на массиве из одного элемента
- querySum на всём диапазоне и на подотрезке
- queryMin/queryMax корректны после build
- update одного элемента — querySum/Min/Max корректно пересчитываются
- rangeUpdate — прибавление к диапазону корректно отражается в querySum
- Lazy propagation: rangeUpdate + query без лишних проталкиваний
- Несколько последовательных rangeUpdate на пересекающихся отрезках
- Запрос на отрезке из одного элемента [i, i]
- Запрос на полном массиве [0, n-1]

## Продвинутые кейсы
- Массив из отрицательных чисел — min/max корректны
- rangeUpdate всего массива + querySum — проверка корректности lazy
- Чередование update и rangeUpdate
- Большой массив (100 000 элементов) — проверка производительности O(log n)
- Граничные индексы: left=0, right=n-1
- Массив из одинаковых элементов
- Массив с Long.MAX_VALUE / Long.MIN_VALUE — проверка переполнения
- rangeUpdate(0, n-1, 0) — не должен менять значения
