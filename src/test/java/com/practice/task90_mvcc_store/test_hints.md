# Подсказки для тестирования: MVCCStore

## Обязательно протестировать
- begin возвращает уникальные монотонно возрастающие txId
- put + commit + get из другой транзакции — значение видимо
- put без commit — значение невидимо для других транзакций
- put + rollback — значение не видимо
- Транзакция видит свои собственные uncommitted записи
- delete помечает ключ как удалённый (get возвращает Optional.empty)
- Write-write конфликт — исключение
- snapshot возвращает только существующие (не удалённые) пары

## Продвинутые кейсы
- Несколько транзакций: tx1 пишет, tx2 читает (не видит), tx1 commit, tx3 читает (видит)
- Перезапись ключа несколькими committed транзакциями — видна последняя
- rollback после нескольких put — все записи откатаны
- gc удаляет старые версии, не ломая активные транзакции
- Операции с несуществующей транзакцией — IllegalStateException
- Повторный commit/rollback — исключение
- Многопоточный доступ: параллельные транзакции корректно изолированы
- delete + put того же ключа в одной транзакции — последняя операция побеждает
