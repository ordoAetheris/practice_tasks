# Подсказки для тестирования: ReadWriteLock

## Обязательно протестировать
- Несколько читателей могут одновременно удерживать блокировку
- Писатель блокируется пока есть активные читатели
- Читатели блокируются пока есть активный писатель
- Два писателя не могут работать одновременно
- releaseRead/releaseWrite корректно освобождают блокировку
- getReadersCount и isWriteLocked возвращают корректные значения

## Продвинутые кейсы
- Предотвращение writer starvation: при ожидающем писателе новые читатели не получают блокировку
- Нет deadlock при чередовании операций чтения и записи
- Корректная работа при множестве конкурентных читателей и писателей (stress test)
- InterruptedException при прерывании ожидающего потока
- Повторный захват блокировки после освобождения работает корректно
- releaseRead без acquireRead — обработка ошибки
