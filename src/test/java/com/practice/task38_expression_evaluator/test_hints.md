# Подсказки для тестирования: ExpressionEvaluatorService

## Обязательно протестировать
- Простое сложение: "2 + 3" = 5.0
- Приоритет операций: "2 + 3 * 4" = 14.0
- Скобки: "(2 + 3) * 4" = 20.0
- Вложенные скобки: "((2 + 3) * (4 - 1))" = 15.0
- Унарный минус: "-5 + 3" = -2.0
- Деление: "10 / 4" = 2.5
- Деление на ноль: "5 / 0" — ожидать ArithmeticException
- Пробелы игнорируются: "2+3" и "2 + 3" — одинаковый результат

## Продвинутые кейсы
- Дробные числа: "2.5 * 4" = 10.0
- Унарный минус в скобках: "-(3 + 2)" = -5.0
- Несколько операций подряд: "1 + 2 + 3 + 4" = 10.0
- Смешанные операции: "2 + 3 * 4 - 6 / 2" = 11.0
- Пустая строка — ожидать IllegalArgumentException
- Несбалансированные скобки: "(2 + 3" — ожидать IllegalArgumentException
- Только число: "42" = 42.0
- Двойной минус: "--5" — как обрабатывать? (определить поведение)
- Выражение с пробелами в начале и конце
