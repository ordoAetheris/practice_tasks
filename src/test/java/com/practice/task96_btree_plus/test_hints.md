# Подсказки для тестирования: BPlusTree

## Обязательно протестировать
- insert + search одного элемента
- Вставка вызывает split листа — данные корректны
- rangeSearch по связным листьям — все элементы в диапазоне, отсортированы
- delete из листа без перебалансировки
- delete с заимствованием у соседа
- delete со слиянием листьев
- min/max корректны
- size увеличивается при insert, уменьшается при delete
- Все данные только в листьях (внутренние узлы — только разделители)

## Продвинутые кейсы
- Вставка дубликата ключа — обновление значения
- rangeSearch на всём диапазоне = полный обход листьев
- rangeSearch пустой диапазон — пустой результат
- Split внутреннего узла: средний ключ перемещается (не копируется)
- Split листа: средний ключ копируется (не перемещается)
- Большое дерево (10 000+ ключей) — rangeSearch эффективен
- Удаление всех ключей — дерево пустое
- height корректна: одноуровневое дерево = 1
- Листья действительно связаны: обход от min до max по next-ссылкам
- Инварианты B+ дерева после каждой операции: ограничения на заполненность узлов
