# Подсказки для тестирования: CsvParserService

## Обязательно протестировать
- parse простого CSV — корректные заголовки и значения
- Поле с запятой внутри кавычек — "hello, world" парсится как одно поле
- Экранирование кавычек — "" внутри поля преобразуется в "
- Пустые поля — пустая строка, не null
- parseWithTypes — Integer, Double, Boolean корректно приводятся
- filter — возвращаются только строки с совпадающим значением
- sort — корректная сортировка по возрастанию и убыванию
- groupBy — корректная группировка
- toCsv — обратная сериализация (round-trip: parse(toCsv(data)) = data)

## Продвинутые кейсы
- CSV с одной строкой (только заголовки) — пустой список данных
- Несовпадение количества полей и заголовков — ожидать ошибку
- parseWithTypes с некорректным значением для типа — ожидать ошибку
- Пустая строка CSV — ожидать ошибку
- Поле с переносом строки внутри кавычек
- toCsv экранирует значения с запятыми и кавычками
- sort по числовой колонке (строковая сортировка: "10" < "2")
- filter по несуществующей колонке
