# Подсказки для тестирования: ThreadSafeQueue

## Обязательно протестировать
- put/take в однопоточном режиме — корректный FIFO порядок
- Блокировка put() при полной очереди (отдельный поток должен разблокировать take())
- Блокировка take() при пустой очереди (отдельный поток должен разблокировать put())
- offer() с таймаутом возвращает false при полной очереди
- poll() с таймаутом возвращает null при пустой очереди
- size(), isEmpty(), isFull() корректны после операций
- Создание очереди с capacity <= 0 бросает исключение

## Продвинутые кейсы
- Множественные producer и consumer потоки одновременно (stress test)
- Корректность при чередовании put/take из разных потоков
- InterruptedException корректно пробрасывается при прерывании ожидающего потока
- Нет deadlock при одновременных операциях
- Порядок FIFO сохраняется при конкурентном доступе
- offer() успевает добавить элемент, если место освобождается до истечения таймаута
