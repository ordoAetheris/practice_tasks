# Подсказки для тестирования: MerkleTree

## Обязательно протестировать
- Построение дерева из чётного и нечётного числа блоков
- getRootHash детерминирован (одни и те же данные → один и тот же хеш)
- getRootHash меняется при изменении любого блока
- getProof возвращает O(log n) шагов
- verify: корректный proof → true
- verify: изменённые данные с правильным proof → false
- verify: подменённый proof → false
- diff двух одинаковых деревьев → пустой список
- diff с изменённым одним блоком → список из одного индекса
- update меняет rootHash

## Продвинутые кейсы
- Дерево из одного элемента — proof пустой, root = hash(data)
- Дерево из степени двойки элементов vs не степень двойки
- diff двух полностью различных деревьев → все индексы
- update + getProof → новый proof валиден для нового rootHash
- Большое дерево (10 000 блоков) — proof длиной ~13 шагов
- Пустой список блоков → исключение
- Нечётное количество: дублирование последнего листа корректно
