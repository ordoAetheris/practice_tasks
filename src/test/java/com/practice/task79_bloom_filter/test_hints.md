# Подсказки для тестирования: BloomFilter

## Обязательно протестировать
- Добавленный элемент всегда найден (mightContain = true)
- Несуществующий элемент — обычно false (нет ложных отрицательных)
- size() корректен после добавлений
- clear() сбрасывает фильтр
- expectedFalsePositiveRate() растёт с заполнением

## Продвинутые кейсы
- Статистическая проверка: при N вставках FPR не превышает заданный порог значительно
- Большое количество элементов (тысячи)
- Разные типы объектов (String, Integer)
- Повторное добавление одного элемента — size не меняется (или меняется, зависит от реализации)
