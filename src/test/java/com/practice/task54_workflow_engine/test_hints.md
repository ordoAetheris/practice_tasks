# Подсказки для тестирования: WorkflowEngineService

## Обязательно протестировать
- defineWorkflow и startInstance — экземпляр создаётся со статусом RUNNING
- completeStep для MANUAL-шага — переход к следующему шагу
- AUTOMATIC-шаг выполняется автоматически после предыдущего
- Линейный процесс (A -> B -> C) — полное прохождение до COMPLETED
- getStatus — корректный статус на каждом этапе
- pause/resume — статус меняется корректно
- getHistory — записи в хронологическом порядке

## Продвинутые кейсы
- completeStep для не текущего шага — ожидать IllegalStateException
- completeStep на приостановленном экземпляре — ожидать ошибку
- Ветвление: из шага два nextSteps с условиями — выбирается нужный
- Процесс из одного AUTOMATIC-шага — сразу COMPLETED
- pause на уже завершённом экземпляре — ожидать ошибку
- resume на не приостановленном — ожидать ошибку
- Запуск несуществующего workflow — ожидать ошибку
- Несколько экземпляров одного workflow — изоляция данных
