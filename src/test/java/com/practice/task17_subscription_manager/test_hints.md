# Подсказки для тестирования: SubscriptionManagerService

## Обязательно протестировать
- Оформление подписки, статус ACTIVE
- Оформление trial, статус TRIAL
- Повторный trial для того же пользователя — исключение
- Продление увеличивает endDate
- Продление отменённой — исключение
- isActive на разные даты (до/после endDate)

## Продвинутые кейсы
- Истёкшая подписка — isActive = false
- Подписки истекающие в ближайшие N дней
- Продление уже истёкшей (но не отменённой)
