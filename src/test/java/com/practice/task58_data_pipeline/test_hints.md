# Подсказки для тестирования: DataPipelineService

## Обязательно протестировать
- FILTER — фильтрация по значению поля, оставляет только подходящие записи
- MAP — добавление/изменение поля в каждой записи
- SORT — сортировка по полю (по возрастанию и убыванию)
- REDUCE — агрегация (sum, count, avg) возвращает одну запись
- GROUP_BY — группировка по полю, корректные группы
- Цепочка стадий: FILTER -> MAP -> SORT — результат корректен
- getIntermediateResults — промежуточные данные после каждой стадии
- validatePipeline — валидный конвейер без ошибок

## Продвинутые кейсы
- Пустые входные данные — пустой результат (не ошибка)
- FILTER убирает все записи — последующие стадии работают с пустым списком
- REDUCE с пустыми данными — определить поведение
- getIntermediateResults до выполнения конвейера — ожидать IllegalStateException
- Конвейер из одной стадии — работает корректно
- Повторное выполнение — промежуточные результаты обновляются
- Несуществующий pipelineId — ожидать ошибку
- MAP с вычисляемым полем (например, fullName = firstName + " " + lastName)
