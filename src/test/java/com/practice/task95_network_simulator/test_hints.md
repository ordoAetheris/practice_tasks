# Подсказки для тестирования: NetworkSimulator

## Обязательно протестировать
- Прямая доставка: A→B по прямому каналу — пакет доставлен
- Маршрутизация: A→C через B (нет прямого канала A→C)
- TTL expiration: маленький TTL, длинный путь → пакет сброшен
- packetLoss = 1.0 → 100% потерь
- packetLoss = 0.0 → 0% потерь
- getRoutingTable содержит записи для всех достижимых узлов
- getStats: sent, received, dropped корректны
- Latency: пакет прибывает через правильное количество шагов

## Продвинутые кейсы
- Кратчайший путь: при нескольких маршрутах выбирается с минимальной latency
- Недостижимый узел (нет пути) → пакет не доставлен
- Добавление нового канала → таблицы маршрутизации обновлены
- packetLoss: статистически ~10% потерь при packetLoss=0.1 (на большой выборке)
- Bandwidth ограничение: очередь пакетов на канале
- Петля маршрутизации — TTL защищает от бесконечного хождения
- Множество одновременных пакетов между разными парами узлов
- avgLatency корректно считается для доставленных пакетов
