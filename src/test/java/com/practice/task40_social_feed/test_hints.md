# Подсказки для тестирования: SocialFeedService

## Обязательно протестировать
- createUser — пользователь создаётся с уникальным id
- createPost — пост привязан к автору, timestamp заполнен
- follow + getFeed — лента содержит посты подписок
- Лента НЕ включает собственные посты пользователя
- Лента отсортирована по времени (новые первыми)
- getFeed с limit — возвращается не более limit постов
- likePost — likesCount увеличивается
- Нельзя подписаться на себя — ожидать IllegalStateException
- Нельзя лайкнуть дважды — ожидать IllegalStateException

## Продвинутые кейсы
- unfollow — посты бывшей подписки пропадают из ленты
- getMutualFollowers — общие подписчики двух пользователей
- getMutualFollowers без общих подписчиков — пустой список
- Повторная подписка на того же пользователя — ожидать исключение
- Пустая лента (нет подписок или подписки без постов)
- Пустое имя пользователя — ожидать исключение
- Пустой контент поста — ожидать исключение
- getUserPosts — только посты указанного пользователя
- Лайк от несуществующего пользователя — ожидать исключение
