# Подсказки для тестирования: DependencyResolverService

## Обязательно протестировать
- addModule без зависимостей — успешно добавляется
- addModule с зависимостью на несуществующий модуль — ожидать ошибку
- resolve — корректный топологический порядок (A->B->C: результат [C, B, A])
- detectCycle — false для ациклического графа
- detectCycle — true при наличии цикла (A->B->C->A)
- removeModule без зависимых — успешно удаляется
- removeModule с зависимыми — ожидать IllegalStateException
- getDependents — корректный список зависимых модулей

## Продвинутые кейсы
- Ромбовидная зависимость (A->B, A->C, B->D, C->D) — D в результате только один раз
- Модуль без зависимостей — resolve возвращает только его
- Глубокая цепочка зависимостей (10+ уровней)
- Добавление модуля, создающего цикл — ожидать ошибку или detectCycle=true
- Несколько независимых подграфов
- resolve несуществующего модуля — ожидать ошибку
- Удаление модуля и повторное добавление
