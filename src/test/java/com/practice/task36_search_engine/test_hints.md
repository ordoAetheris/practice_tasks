# Подсказки для тестирования: SearchEngineService

## Обязательно протестировать
- Добавление документа и поиск по одному слову — документ найден
- search (AND) — документ содержит ВСЕ слова запроса
- search (AND) — документ не содержит одно из слов — не найден
- searchAny (OR) — документ содержит хотя бы одно слово — найден
- Регистронезависимый поиск — "Java" находит "java"
- Ранжирование по TF — документ с большей частотой слова выше
- Подсветка — слова из запроса в [квадратных скобках]
- Удаление документа — после удаления не находится поиском

## Продвинутые кейсы
- Пустой запрос — пустой результат
- Поиск слова, которого нет ни в одном документе — пустой результат
- Дубликат id при добавлении — ожидать исключение
- Подсветка сохраняет оригинальный регистр слов
- Документ с повторяющимися словами — корректный TF
- Удаление и повторное добавление документа с тем же id
- Знаки препинания в тексте не влияют на поиск
- Многословный запрос — ранжирование по суммарному TF
