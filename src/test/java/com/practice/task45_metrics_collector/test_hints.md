# Подсказки для тестирования: MetricsCollectorService

## Обязательно протестировать
- record и getAvg — корректное среднее значение
- getMin/getMax — корректные экстремумы
- getPercentile — p50 для [1,2,3,4,5] = 3, p95 для большой выборки
- getCount — корректное количество записей за окно
- Скользящее окно — старые записи не учитываются
- getAllMetricNames — возвращает все имена зарегистрированных метрик
- Несколько метрик одновременно — изоляция данных

## Продвинутые кейсы
- Пустое окно — getAvg/getMin/getMax бросают NoSuchElementException
- Несуществующая метрика — ожидать ошибку или 0
- percentile = 0 и percentile = 100 — граничные случаи
- percentile < 0 или > 100 — ожидать IllegalArgumentException
- Одно значение за окно — avg = min = max = это значение
- Запись с timestamp в будущем — корректная обработка
- Большое количество записей — производительность
