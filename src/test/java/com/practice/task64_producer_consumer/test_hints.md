# Подсказки для тестирования: ProducerConsumerService

## Обязательно протестировать
- produce/consume в однопоточном режиме — FIFO порядок
- Backpressure: producer блокируется при полном буфере
- Consumer блокируется при пустом буфере
- getStats() корректно считает produced и consumed
- Один producer + один consumer корректно передают все данные
- shutdown() завершает потоки корректно

## Продвинутые кейсы
- Множество producer + множество consumer — все элементы обработаны ровно один раз
- Graceful shutdown: элементы в буфере дообрабатываются перед остановкой
- Статистика корректна при конкурентном доступе
- Producer, возвращающий null, корректно останавливается
- Нет потерянных элементов при высокой нагрузке
- Корректная работа при неравной скорости producer и consumer
